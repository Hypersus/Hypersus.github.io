521. 最长特殊序列 Ⅰ

题目

给你两个字符串 a 和 b，请返回 这两个字符串中 最长的特殊序列 的长度。如果不存在，则返回 -1 。

「最长特殊序列」 定义如下：该序列为 某字符串独有的最长子序列（即不能是其他字符串的子序列） 。

字符串 s 的子序列是在从 s 中删除任意数量的字符后可以获得的字符串。

例如，"abc" 是 "aebdc" 的子序列，因为删除 "aebdc" 中斜体加粗的字符可以得到 "abc" 。 "aebdc" 的子序列还包括 "aebdc" 、 "aeb" 和 "" (空字符串)。

题解

脑筋急转弯，判断两个字符串是否相等，不等的话则返回两个字符串中较长的那个字符串的长度值，因为长字符串必定有一个子序列(即自身)不是短字符串的子序列

258. 各位相加

题目

给定一个非负整数num，反复将各个位上的数字相加，直到结果为一位数。返回这个结果。

题解

实际是找非负整数num的树根，使用非负整数的神奇的性质可以在时间复杂度O(1)内解决，甚至只用一行代码。。。

对于非负整数num，假设其长度为n，从最高位到最低位的数字分别为\(d_{n-1}...d_0\)则有 \[\begin{aligned} num &= \sum_{i=0}^{n-1}d_i*10^i \\ &=\sum_{i=0}^{n-1}d_i*(10^i-1+1) \\ &= \sum_{i=0}^{n-1}d_i*(10^i-1) + \sum_{i=0}^{n-1}d_i\end{aligned}\]

注意到\((10^i-1)\)是9的整数倍，因此有 \[ num \%9 = (\sum_{i=0}^{n-1}d_i) \% 9 \] 此时\(\sum_{i=0}^{n-1}d_i\)不一定是个位数，仅仅是num各位数的相加结果，假设\(num=\sum_{i=0}^{m-1}d_i\)，其长度为m，则可以重复上述过程直到某个\(\sum_{i=0}^{p-1}d_i\)为个位数。 \[ num \%9 = (\sum_{i=0}^{n-1}d_i) \% 9=(\sum_{i=0}^{m-1}d_i) \% 9=...=(\sum_{i=0}^{p-1}d_i) \% 9 \] 此时还只找到了\(num\%9\)和\((\sum_{i=0}^{p-1}d_i) \% 9\)的关系，并不等于直接找到了\(\sum_{i=0}^{p-1}d_i\)，接下来继续分析

  • 当\(num\neq0\)时，各位上的非负整数相加结果一定非负即\(\sum_{i=0}^{p-1}d_i\in(0,9]\)
    • \(\sum_{i=0}^{p-1}d_i-1\in[0,9)\)
    • 对\(\sum_{i=0}^{p-1}d_i\in[0,9)\)，有\(\sum_{i=0}^{p-1}d_i \% 9 = \sum_{i=0}^{p-1}d_i\)
    • 对\(\sum_{i=0}^{p-1}d_i\in(0,9]\)，有\(\sum_{i=0}^{p-1}d_i=\sum_{i=0}^{p-1}d_i-1+1=(\sum_{i=0}^{p-1}d_i-1)\%9+1\)
    • \((\sum_{i=0}^{p-1}d_i-1)\%9=(\sum_{i=0}^{p-1}d_i\%9-1\%9)\%9=(num\%9-1\%9)\%9=(num-1)\%9\)
    • 于是可以得出当\(num\neq0\)时，\(\sum_{i=0}^{p-1}d_i=(num-1)\%9+1\)

剑指 Offer 24. 反转链表

题目：

定义一个函数，输入一个链表的头节点，反转该链表并输出反转后链表的头节点。

示例:

输入: 1->2->3->4->5->NULL 输出: 5->4->3->2->1->NULL

题解：

可以递归，一开始以为因为返回的是子过程反转链表的头结点，所以添加新结点时需要从新的头结点开始遍历。但是这种想法忽略了可以通过head.Next访问原链表中当前结点的下一个结点，即子过程反转链表的尾结点。于是可以通过head.Next.Next = head修改子过程反转链表的尾结点。

迭代简单点，分别存储3个结点的地址设为prev cur next，cur初始化为head结点，prev初始化为nil，这样原链表就变成了nil head->....->tail->nil，相当于用一个长度为3的滑窗进行遍历，每一次遍历修改cur.Next=prev，然后滑窗沿着原链表滑动，直到cur为tail执行最后一次指针修改，for循环判断条件为cur!=nil

剑指 Offer 53 - I. 在排序数组中查找数字

题目

统计一个数字在排序数组中出现的次数。其中nums是非递减数组

题解

方法一：

普通遍历，从头遍历到尾，直到某个元素大于target跳出循环。

pros: 干净又卫生！！代码简洁易读

cons: 时间复杂度较高，没有利用到数组非递减的性质，但是代码简单啊~

方法二：

估计考这题的目的就是让写二分查找，那么由于target要么不在给定数组里，要么存在且连续排列，统计target次数 = rightmost_index - leftmost_index + 1，按照这个思路只要找到leftmost_index和rightmost_index，写两个循环按照二分法查找leftmost_index和rightmost_index

于是关键在于leftmost_index和rightmost_index的两个循环

对于寻找leftmost_index的循环，两个索引left和right初始需要分别置为0和len(nums)，由于整除操作是向下取整，因此相比于right索引，mid会更靠近left索引，不会出现下标越界。基本思路是left=mid+1不断靠近第一个大于等于target的索引，right=mid不断靠近left，最终让left=right

对于寻找rightmost_index的循环，思路类似，但是需要left和right的初始值分别置为-1和len(nums)-1，并且为了让mid靠近right索引，需要给mid赋值mid = (left + right) >> 1 + 1

题解

这题涉及到深拷贝（deep copy）的问题，即对要拷贝的对象，在内存中开辟一个新的空间，并将该对象（结构体）的全部内容保存在这个新开辟的内存空间里。浅拷贝与深拷贝相对，通常是对引用/指针的拷贝。

难点在由于random指针的存在使得不能按照遍历的顺序来创建结点

如果只有next指针，则可以

为了解决这个问题，有两个思路

方法一：回溯+哈希表

维护一个哈希表map[*Node]*Node，存放原链表结点地址和新链表结点地址的映射，如果对于给定的原链表结点，哈希表中不存在映射，说明对应的新链表结点还没有创建，于是需要创建新结点，创建新结点后把新结点的地址存入该哈希表中，于是在为新结点的next指针和random指针赋值时，可以指针指向的原链表结点地址查询该哈希表得到对应的新链表结点地址。

deep copy vs shallow copy

similarity: both a method of object copy

difference:

  • In shallow copy a new object B is created, and the fields values of A are copied over to B. This is also known as a field-by-field copy, field-for-field copy, or field copy. If the field value is a reference to an object (e.g., a memory address) it copies the reference, hence referring to the same object as A does, and if the field value is a primitive type it copies the value of the primitive type. In languages without primitive types (where everything is an object), all fields of the copy B are references to the same objects as the fields of original A. The referenced objects are thus shared, so if one of these objects is modified (from A or B), the change is visible in the other. Shallow copies are simple and typically cheap, as they can usually be implemented by simply copying the bits exactly.
  • An alternative is a deep copy, meaning that fields are dereferenced: rather than references to objects being copied, new copy objects are created for any referenced objects, and references to these placed in B. The result is different from the result a shallow copy gives in that the objects referenced by the copy B are distinct from those referenced by A, and independent. Deep copies are more expensive, due to needing to create additional objects, and can be substantially more complicated, due to references possibly forming a complicated graph.

Some facts about slices

  • When declared but not initialized, the value of the slice s (i.e. the value of pointer) is 0x0
  • When initialized empty the pointer is NOT 0x0
  • When specified using [low_bound:high_bound], the value of s is the address of low_bound element of the underlying array in memory.
  • Use built-in make() to initialize the slice with length and capacity specified.
