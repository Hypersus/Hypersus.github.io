packages and modules

Posted on 2022-02-03 In golang

modules: every module contains a go.mod file in which the module name is declared and xxxx.go file in which package name is declared.

Q: It seems that a module could only contain a single package and the package name should be maintained the same as the basename of module name ?

A: Effectively, golang is able to import different packages in the same module (take local module example.com/greetings as an example)

  • First, in local directory greetings, a go.mod is created by typing the command go mod init example.com/greetings so we get the module name example.com/greetings.
  • Now if we want to import 2 different packages in the same module example.com/greetings , we need 2 different .go files in the first place and then create an exclusive folder for each of them, the name of the folder doesn't really matter since it was only used for locating the path of the package and the .gofile name doesn't really matter. Given we had 2 packages naming package1 and package2 declared in 2 files xxxx1.go and xxxx2.go . Create folder1 in greetings and move file xxxx1.go into it and xxxx2.go into folder2.
  • If we want to import the abovementioned package package1 and package2 in an external file. Write import "example.com/greetings/xxxx1" to import package1 and the same way to import package2
  • run go mod tidy in the caller directory

Conclusion: while importing custom local or online packages in golang, we need to specify a path that contains 1 package ONLY and use the declared package name to access its functions

单精度float(32 bit)

计算公式

\[ value=(-1)^{b_{31}}\cdot 2^{(b_{30}b_{29}...b_{23})_2 -127}\cdot (1+\sum_{i=1}^{23}b_{23-i}2^{-i}) \]

其中\((b_{30}b_{29}b_{23})_2\)全0和全1保留作为其他用处，即\(2^{(b_{30}b_{29}...b_{23})_2 -127}\)的取值集合为\(\{2^{-126},...,2^{127}\}\)

表

其中指数部分全0和全1的情况指向的数字如下表：

Exponent fraction = 0 fraction≠0 Equation
\(00_H=0000,0000_2\) ±zero subnormal number \(value=(-1)^{b_{31}}\cdot 2^{-126}\cdot (0+\sum_{i=1}^{23}b_{23-i}2^{-i})\)
\(01_H,...,FE_H\) normal value normal value \(value=(-1)^{b_{31}}\cdot 2^{(b_{30}b_{29}...b_{23})_2 -127}\cdot (1+\sum_{i=1}^{23}b_{23-i}2^{-i})\)
\(FF_H=1111,1111_2\) ±infinity NaN

半精度float(16 bit)

仿照单精度的计算公式，有： \[ value=(-1)^{b_{15}}\cdot 2^{(b_{14}b_{13}...b_{10})_2 -15}\cdot (1+\sum_{i=1}^{10}b_{10-i}2^{-i}) \]

Exponent fraction = 0 fraction≠0 Equation
\(0,0000_2\) ±zero subnormal number \(value=(-1)^{b_{15}}\cdot 2^{-14}\cdot (0+\sum_{i=1}^{10}b_{10-i}2^{-i})\)
\(0,0001_2,...,1,1110_2\) normal value normal value \(value=(-1)^{b_{15}}\cdot 2^{(b_{14}b_{13}...b_{10})_2 -15}\cdot (1+\sum_{i=1}^{10}b_{10-i}2^{-i})\)
\(FF_H=1111,1111_2\) ±infinity NaN

Pros:

  • 减少显存占用
  • 加快训练和推断

Cons:

  • 溢出错误（Grad Overflow/Underflow）:FP16的动态范围狭窄导致上溢出和下溢出——由于激活函数梯度小更容易发生下溢出
  • 舍入误差（Rounding Error）:梯度值过小，小于FP16的最小间隔，导致反向传播时无法更新梯度如下图

​ Tips:FP16和FP32的最小间隔随着exponent项的改变而动态改变，exponent大时相应最小间隔绝对值也大(对1000来说0.001和0.1差别不大)，fraction项的位数n决定了在normal的各区间内最小间隔为区间下界的\(2^{-n}\)倍

Solutions

  • 混合精度训练(Mixed Precision):在内存中用FP16做储存和乘法从而加速计算，用FP32做累加避免舍入误差，有效解决舍入误差的问题
  • 损失放大(Loss Scaling):激活函数梯度值仍然太小，造成下溢出，Loss Scaling在做的是
    • 反向传播前，将Loss的改变量手动增大\(2^k\)倍，保证中途不会溢出
    • 反向传播后，将权重梯度缩小\(2^k\)倍

关于在博客中插入图像和音频

插入图像

根据前述插入图像的方法，需要为每一篇博客新建一个文件夹，文件夹名字和.md名字保持一致，才可以使用markdown原生的插入图片方法即![](url)插入图片，这样子的插入图片方法不易于管理。

替代方案为：在hexo的安装路径下的blog\source\目录中，新建一个img文件夹用于保存所有图片，这样子使用hexo g时hexo会为该img文件夹建立一个在blog\public\目录下的映射，于是可以使用![](/img/example.png)来插入图片

cons：用这样的路径插入图片，在本地编辑md时无法预览图片也就无法知道路径是否写错

pros：易于管理

插入音频

以上插入图像的替代方法主要是在探索hexo中插入音频过程中发现的，在markdown中插入音频似乎只能用HTML标签，使用HTML标签插入音频的方法为<audio src="" controls="true"></audio>。和上述插入图像不同，假设存在一篇博客文件名为xxxx.md，当使用![](xxxx/example.png)插入图像，hexo g会在其生成的index.html中对路径进行转换，使用相对于public目录的相对路径来填充<img>标签中的src属性。

查看public目录的结构可以发现，路径的根目录/即对应public目录，使用该路径在本地的public目录中定位图片所在位置，可以找到

至此就可以得出结论了

任何保存在xxxx文件夹中的资源（图片/音频/视频）在hexo g之后都会被映射到public/timestamp/xxxx路径下，其中index.html对应.md文件，其余相关资源被保存在同路径下。

所以如果插入音频时也像插入图像一样指定src="xxxx/example.wav，由于hexo g不会对<audio>标签中的src属性进行修改，所以在index.html中资源访问路径仍然是xxxx/example.wav，但是index.html并没有同级目录xxxx（有上级目录xxxx），所以资源访问会出错，也就找不到音频文件了。

解决方法：

在source路径下新建audio目录，把所有音频文件存放在该目录中，使用src="/audio/example.wav"进行资源访问
